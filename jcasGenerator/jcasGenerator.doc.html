JcasGenerator

Je vais vous expliquer comment j’ai réaliser la génération de code Jcas.

Fonctionnement

Le programme .jar doit être placé dans le dossier de test.

java -jar jcas.jar

Une fois lancé, le programme demande le nombre de fichier de tests qui doit être écrit.
Ensuite il faut spécifier juste ou faux pour spécifier si l’ensemble des programmes créés soit faux lexicalement et syntaxiquement.
Enfin il ne faudra plus que lancer les scripts qui feront le travail.

Architecture

Le programme génère un ensemble de variables aléatoires et un ensemble de types qu’il y associe.
Ensuite il prend en compte les variables générés et utilise parmis les fonctions write, read, if, while le contenu du programme. L’implémentation de fonctions imbriquées n’est pas prise en compte. Toutefois le type des variables est prise en compte lors des affections et des conditions.
Les commentaires sont également générés en nombre aléatoire.

Programme faux

En spécifiant faux, le programme va insérer des morceaux de programmes à partir d’un programme juste pour le fausser. On aura ainsi plusieurs cas possibles de faux programmes.
